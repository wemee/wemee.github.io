最近我常在思考科技發展的一個有趣悖論：**我們創造了越來越強大的軟體，但為了使用它們，人類付出的學習成本卻越來越高。**

這促使我重新審視軟體介面（Interface）的演進史。如果我們把「人」與「運算結果」之的距離看作一道鴻溝，那麼這段歷史，其實就是不斷填平這道鴻溝的過程。

我認為這個演進可以分為三個階段。

## 第一階段：人適應機器 (The Era of Adaptation)

在過去，軟體被開發出來後，責任就轉移到了使用者身上。

![複雜的說明書與介面](/images/blog/evolution/past.webp)

我們有了功能強大的 Photoshop、Excel 或是各種企業級 ERP 系統。它們無所不能，但也極其難用。我們必須：
*   閱讀厚厚的操作手冊。
*   記憶各種快捷鍵。
*   理解工程師設計的「邏輯」（為什麼設定在這個選單？為什麼要先點A才能點B？）。
*   面對無止盡的 Config 設定檔。

在這個階段，**「門檻」是人類的學習成本**。我們花費大量時間去適應工具，把自己的大腦訓練成符合軟體邏輯的形狀。這是一個「人去配合軟體」的時代。

## 第二階段：Agent 作為中介 (The Agentic Shift)

現在，或是最近的未來，我們正在經歷一場巨大的典範轉移。AI Agent 的出現改變了遊戲規則。

![AI Agent 操作介面](/images/blog/evolution/present.webp)

軟體本身可能依然很難用，甚至為了容納更多功能變得**更難用**（更多的 API 參數、更靈活但繁瑣的設定）。但這已經不再重要了，因為**軟體不再是設計給人用的**。

在這個階段：
1.  **人類的角色**：退居幕後，只負責用自然語言下達「意圖 (Intent)」。
2.  **Agent 的角色**：充當「萬能操作員」。它去讀那本幾千頁的手冊，它去記住所有的快捷鍵，它去幫你點擊那些隱藏在十層選單下的按鈕。

我們開始看到像 Browser Agent 這樣的技術，它們能像人一樣操控瀏覽器；或是能夠操控電腦畫面的 OS Agent。軟體的複雜度被 Agent 封裝了起來。介面（Interface）雖然存在，但對人類來說已經逐漸透明化。

## 第三階段：軟體即時生成 (Software on Demand)

這才是最讓我興奮，也最令人感慨的終局。

如果 Agent 已經強大到能理解所有意圖，並能精準操作電腦，那我們**還需要預先開發好的軟體嗎？**

![意圖即代碼的未來](/images/blog/evolution/future.webp)

想像一下，未來你不需要去下載一個「修圖軟體」來把照片變亮。你只需要告訴 AI：「幫我把這張照片調亮，去掉背景雜訊。」

在那一瞬間，AI 並不是去打開某個現成的 App，而是：
1.  **理解需求**。
2.  **即時撰寫 (Just-in-Time)** 出一段專屬於這個任務的小程式（Script）。
3.  **執行**並回傳結果。
4.  任務結束，這段程式隨即銷毀。

就像我們現在看到的 **Gemini CLI**, **Claude Code**, 或 **OpenAI Codex** 的雛形。未來的軟體將不再是固定的「產品」，而是隨用隨丟的「液態工具」。

軟體內部甚至不需要有功能。因為當你需要時，自然語言會指揮 AI Agent **現場開發**出這個功能。

## 結語

從「人去學軟體」，到「Agent 幫人操作」，最後是「軟體隨意圖而生」。

這條演進之路的終點，是**介面的徹底消亡**。當自然語言成為唯一的介面，人類終於可以從繁瑣的操作中解放，將精力回歸到最本質的事物上——**創造力與意圖**。

這不僅是科技的進步，更是人機關係的一場溫柔革命。
